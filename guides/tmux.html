$ tmux field guide

A practical reference for persistent terminal sessions — from first launch to a fully configured setup.

What tmux actually is

tmux is a terminal multiplexer. It runs a server on your machine that keeps your shell processes alive regardless of whether you're connected. Three nested concepts to know:

S
Session
A named workspace. You detach from it, everything keeps running. You can have multiple sessions (e.g. "work", "server", "personal").
W
Window
Like a tab inside a session. Each window fills the full terminal. Switch between them with a number.
P
Pane
A split within a window. Multiple panes share one window — useful for editor + terminal side by side.

The prefix key

Every tmux command starts with a prefix. The default is Ctrl+b. Press it, release it, then press the command key. You'll change this in config.

Tip: Most people remap prefix to Ctrl+a (screen-style) or Ctrl+Space. The config section covers this.

Start here

~ $ tmux new -s main
# creates a session named "main" and enters it

~ $ tmux ls
main: 1 windows (created Thu Apr 2 09:00:00 2026)

~ $ tmux attach -t main
# reattach after detaching or reconnecting via SSH

Resize in larger increments

The actual daily loop

How a typical session goes once you have tmux set up:

1
SSH in, attach or create
tmux attach || tmux new -s main — one command that attaches if a session exists, or creates one if not.
2
Name your windows meaningfully
Window 1: editor. Window 2: server/logs. Window 3: scratch. Name them with ^b ,.
3
Split for context
Open a second pane in the same window when you need to see two things at once. Zoom in/out as needed.
4
Detach freely
^b d and disconnect. Everything is still running on the server. Come back hours later, pick up exactly where you left off.
5
Use named sessions for projects
Long-running work gets its own session: tmux new -s greywellness. Switch between sessions with ^b s.

Quick shell alias

Add this to your .zshrc or .bashrc:

.zshrc 
# Attach to existing session or create new one
alias t='tmux attach 2>/dev/null || tmux new -s main'

# Jump to a named session quickly
alias tw='tmux new-window'
alias ts='tmux switch-client -t'

Copy mode

Copy mode lets you scroll through terminal history and copy text without a mouse. Enter it with ^b [. With vim keys configured it becomes intuitive fast.

^b [
Enter copy mode
q / Esc
Exit copy mode
Space
Start selection (vi mode)
Enter
Copy selection and exit
^b ]
Paste from tmux buffer
/ or ?
Search forward / backward in history
g / G
Jump to top / bottom of history
Note: Set set-window-option -g mode-keys vi in config to use vim motions in copy mode. Then v to start selection, y to yank.

~/.tmux.conf recommended

Create this file and run tmux source ~/.tmux.conf or restart tmux to apply.

~/.tmux.conf 
# ── PREFIX ────────────────────────────────────────────
# Remap prefix to Ctrl+a (easier than Ctrl+b)
unbind C-b
set -g prefix C-a
bind C-a send-prefix

# ── GENERAL ───────────────────────────────────────────
set -g default-terminal "tmux-256color"
set -ga terminal-overrides ",*256col*:Tc"  # true color
set -g history-limit 50000      # scrollback buffer
set -g escape-time 0            # no delay after Esc (vim fix)
set -g focus-events on           # pass focus events to apps
set -g mouse on                  # mouse scrolling + pane clicking
set -g renumber-windows on       # keep window numbers sequential
set -g base-index 1              # windows start at 1 not 0
setw -g pane-base-index 1        # panes start at 1 not 0

# ── VIM KEYS ──────────────────────────────────────────
setw -g mode-keys vi             # vim in copy mode

# Navigate panes with vim keys
bind h select-pane -L
bind j select-pane -D
bind k select-pane -U
bind l select-pane -R

# Resize panes with vim keys (repeatable)
bind -r H resize-pane -L 5
bind -r J resize-pane -D 5
bind -r K resize-pane -U 5
bind -r L resize-pane -R 5

# ── SPLITS ────────────────────────────────────────────
# More intuitive split bindings, open in current path
bind | split-window -h -c "#{pane_current_path}"
bind - split-window -v -c "#{pane_current_path}"
unbind '"'
unbind %

# New window also opens in current path
bind c new-window -c "#{pane_current_path}"

# ── COPY MODE ─────────────────────────────────────────
bind -T copy-mode-vi v send-keys -X begin-selection
bind -T copy-mode-vi y send-keys -X copy-selection-and-cancel
bind -T copy-mode-vi C-v send-keys -X rectangle-toggle

# ── CONVENIENCE ───────────────────────────────────────
# Reload config with prefix+r
bind r source-file ~/.tmux.conf \; display "Config reloaded"

# Synchronize panes (type in all panes at once)
bind S setw synchronize-panes
Key wins from this config: | and - for splits (much more intuitive than % and "), new panes/windows open in your current directory, and prefix+r reloads config without restarting.

Status bar

The status bar at the bottom of tmux shows sessions, windows, and system info. Here's what a clean setup looks like:

⬡ main 1:zsh 2:nvim* 3:logs
Thu Apr 2 09:41
~/.tmux.conf — status bar 
# ── STATUS BAR ────────────────────────────────────────
set -g status on
set -g status-position bottom
set -g status-interval 5           # refresh every 5 seconds
set -g status-left-length 30
set -g status-right-length 50

# Colors (adjust to taste)
set -g status-style 'bg=#1e2030 fg=#c8d3f5'
set -g status-left '#[fg=#82aaff,bold] #S  '
set -g status-right '#[fg=#636da6] %a %b %-d  #[fg=#c8d3f5]%H:%M '

# Window styles
set -g window-status-format '#[fg=#636da6] #I:#W '
set -g window-status-current-format '#[fg=#82aaff,bold] #I:#W '

# Pane borders
set -g pane-border-style 'fg=#2d3f76'
set -g pane-active-border-style 'fg=#82aaff'

TPM — Plugin Manager

tmux Plugin Manager (TPM) adds a plugin ecosystem. Install it once, then manage plugins from your config.

~ $ git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm
# Then add the TPM block to your .tmux.conf (below)
# Inside tmux: prefix + I to install plugins
# prefix + U to update, prefix + alt+u to uninstall
~/.tmux.conf — add at the bottom 
# ── PLUGINS (TPM) ─────────────────────────────────────
set -g @plugin 'tmux-plugins/tpm'

# Restore sessions after reboot
set -g @plugin 'tmux-plugins/tmux-resurrect'
set -g @plugin 'tmux-plugins/tmux-continuum'
set -g @continuum-restore 'on'       # auto-restore on start
set -g @continuum-save-interval '15' # save every 15 min

# Better search in copy mode
set -g @plugin 'tmux-plugins/tmux-copycat'

# Open files/URLs from copy mode (o to open)
set -g @plugin 'tmux-plugins/tmux-open'

# Smarter pane switching (works with vim splits too)
set -g @plugin 'christoomey/vim-tmux-navigator'

# Initialize TPM — KEEP THIS LINE LAST
run '~/.tmux/plugins/tpm/tpm'

Plugin highlights

tmux-resurrect
Save and restore sessions across reboots
prefix+Ctrl+s to save, prefix+Ctrl+r to restore
tmux-continuum
Auto-saves resurrect state on a timer
Set and forget. Your layout survives a reboot.
vim-tmux-navigator
Ctrl+h/j/k/l moves across vim splits AND tmux panes
Requires matching vim plugin. Seamless.
Note: For your Mac mini setup, tmux-resurrect is especially worth it — your window layout and pane positions survive a reboot without having to remember what you had open.
